# 简历筛选与自动投递工具 - PRD 与技术方案

## 一、产品概述

### 1.1 产品定位
一款面向求职者的智能化招聘辅助工具，通过自动化技术帮助用户在多个招聘平台进行岗位筛选、简历匹配和自动投递，提升求职效率。

### 1.2 目标用户
- 正在主动求职的职场人士
- 希望同时管理多个招聘平台投递进度的用户
- 需要根据不同岗位类型定制简历的求职者

### 1.3 核心价值
- **效率提升**：自动化处理繁琐的投递工作
- **精准匹配**：AI 辅助筛选符合要求的岗位
- **统一管理**：多平台投递状态集中查看
- **智能定制**：根据岗位类型自动选择合适简历版本

## 三、非功能需求

### 3.1 安全性
- 用户密码采用 bcrypt 加密存储
- 招聘平台凭证采用 AES-256 加密
- 所有 API 请求需 JWT 认证
- HTTPS 强制加密传输
- 敏感操作需二次验证

### 3.2 可用性
- 系统可用率 ≥ 99%
- 计划内维护需提前通知
- 数据每日备份

### 3.3 性能要求
- 页面加载时间 < 3s
- API 响应时间 < 500ms（非爬虫接口）
- 支持并发用户数 ≥ 100

### 3.4 可扩展性
- 模块化设计，便于新增招聘平台
- 配置化筛选规则引擎
- 支持插件式 AI 服务扩展

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **前端框架** | Vue 3 + TypeScript | 生态成熟，开发效率高，适合后台管理类应用 |
| **UI 组件库** | Element Plus | Vue 3 生态最成熟的 UI 库 |
| **前端构建** | Vite | 快速开发体验 |
| **后端框架** | FastAPI (Python) | 异步支持好，适合爬虫和 AI 集成，开发效率高 |
| **数据库** | PostgreSQL | 稳定可靠，支持 JSON 类型 |

### 4.3 数据库设计

#### 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 招聘平台账号表
CREATE TABLE platform_accounts (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    platform VARCHAR(50) NOT NULL,  -- boss, maimai, liepin, lagou
    account_identifier VARCHAR(255), -- 账号标识
    credentials_encrypted TEXT,      -- 加密的登录凭证
    cookies_encrypted TEXT,          -- 加密的 Cookie
    login_status VARCHAR(20) DEFAULT 'active', -- active, expired, blocked
    last_sync_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 简历表
CREATE TABLE resumes (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    name VARCHAR(255) NOT NULL,
    file_path VARCHAR(500),
    file_type VARCHAR(20),
    tags TEXT[],                     -- 标签数组
    is_default BOOLEAN DEFAULT FALSE,
    parsed_content JSONB,            -- AI 解析后的结构化内容
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 筛选规则表
CREATE TABLE filter_rules (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    name VARCHAR(255) NOT NULL,
    salary_min INTEGER,
    salary_max INTEGER,
    experience_min INTEGER,
    experience_max INTEGER,
    locations TEXT[],
    remote_accepted BOOLEAN DEFAULT FALSE,
    skills_required TEXT[],
    industries TEXT[],
    company_scale TEXT[],           -- 规模要求
    min_match_score INTEGER DEFAULT 60,
    keywords_include TEXT[],
    keywords_exclude TEXT[],
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 岗位表
CREATE TABLE jobs (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    platform VARCHAR(50) NOT NULL,
    platform_job_id VARCHAR(255),    -- 平台岗位ID
    title VARCHAR(500),
    company VARCHAR(255),
    salary_min INTEGER,
    salary_max INTEGER,
    location VARCHAR(255),
    experience_required VARCHAR(100),
    education_required VARCHAR(100),
    jd_content TEXT,
    jd_url VARCHAR(500),
    match_score INTEGER,
    status VARCHAR(50) DEFAULT 'pending', -- pending, applied, viewed, interview, rejected
    applied_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 投递记录表
CREATE TABLE applications (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    job_id UUID REFERENCES jobs(id),
    resume_id UUID REFERENCES resumes(id),
    platform VARCHAR(50) NOT NULL,
    status VARCHAR(50) DEFAULT 'applied',
    applied_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    response_at TIMESTAMP,
    notes TEXT
);

-- 投递任务表
CREATE TABLE delivery_tasks (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    rule_id UUID REFERENCES filter_rules(id),
    scheduled_date DATE NOT NULL,
    status VARCHAR(50) DEFAULT 'pending', -- pending, running, completed, failed
    jobs_found INTEGER DEFAULT 0,
    jobs_applied INTEGER DEFAULT 0,
    error_message TEXT,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.4 招聘平台接入方案

各平台均无官方开放 API，需通过浏览器自动化模拟登录和操作：

| 平台 | 登录方式 | 接入难度 | 注意事项 |
|------|----------|----------|----------|
| **BOSS直聘** | 扫码登录 | 中 | 需要处理扫码等待，Cookie 有效期约 7 天 |
| **脉脉** | 手机验证码/扫码 | 中 | 验证码需人工介入 |
| **猎聘** | 账号密码/扫码 | 低 | 相对宽松 |
| **拉勾** | 账号密码/扫码 | 低 | 相对宽松 |

**技术实现：**
1. 使用 Playwright 控制 Chromium 浏览器
2. 香次登录引导用户扫码/验证，保存 Cookie
3. 后续操作使用 Cookie 保持登录状态
4. 监测登录失效，通知用户重新登录

**反爬应对策略：**
- 随机化操作间隔（3-10秒）
- 模拟人类鼠标移动轨迹
- 使用代理 IP 池（可选）
- 设置每日操作上限

### 4.5 AI 集成方案

**使用场景：**
1. **简历解析**：提取技能、工作经历、教育背景
2. **JD 分析**：提取岗位要求关键词
3. **匹配度计算**：计算简历与 JD 的匹配分数
4. **简历版本推荐**：根据 JD 推荐最合适的简历版本

**推荐方案：DeepSeek API**
- 价格低廉（约 ¥1/百万 tokens）
- 中文能力强
- API 兼容 OpenAI 格式

**备选方案：**
- 阿里通义千问
- 百度文心一言
- 本地部署 Ollama + Qwen

## 五、用户流程

### 5.1 首次使用流程

```
注册账号 → 邮箱验证 → 设置个人信息 → 绑定招聘平台账号(扫码)
    → 上传简历 → 配置筛选规则 → 启用自动投递
```

### 5.2 日常使用流程

```
登录系统 → 查看仪表盘(投递统计) → 查看投递记录/状态更新
    → (可选)调整筛选规则 → (可选)手动投递特定岗位
```

### 5.3 自动投递流程

```
定时任务触发 → 检查平台登录状态(失效则通知)
    → 登录平台 → 搜索符合规则的岗位
    → 拉取岗位详情 → AI 计算匹配度
    → 筛选达标岗位 → 选择对应简历版本
    → 执行投递(控制间隔) → 记录结果
    → 更新统计数据 → 发送通知
```

## 六、风险评估与应对

### 6.1 账号安全风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 平台账号被封禁 | 高 | 控制操作频率、模拟人工行为、设置每日上限 |
| 凭证泄露 | 高 | AES-256 加密存储、传输 HTTPS |
| 跨平台数据泄露 | 中 | 数据隔离、定期安全审计 |

### 6.2 平台反爬风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| IP 被封 | 中 | 代理 IP 池、降低请求频率 |
| 账号被风控 | 高 | 随机化操作、模拟人类行为 |
| 页面结构变更 | 中 | 监控页面变化、及时更新爬虫 |
| 验证码/滑块 | 中 | 手动介入、OCR 识别 |

### 6.3 法律合规风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 用户协议违规 | 中 | 用户自行承担风险、明确免责声明 |
| 数据使用合规 | 中 | 仅用于用户个人投递、不存储敏感个人信息 |

### 6.4 建议免责声明

在用户注册时明确告知：
- 本工具仅供个人求职辅助使用
- 用户需自行承担使用自动化工具可能带来的账号风险
- 平台有权封禁使用自动化工具的账号

## 七、开发计划建议

> **说明：** 根据用户反馈，采用「本地开发优先」策略，邮件通知和 AI 功能暂缓，先实现核心投递功能。

### 7.1 MVP 功能范围（第一阶段）- 本地开发

**目标：** 在本地环境搭建可运行的基础版本，验证核心流程

**核心功能：**
- [ ] 用户注册/登录（简化版，无邮箱验证）
- [ ] 单平台支持（BOSS直聘 - 先实现一个平台验证方案可行性）
- [ ] 账号绑定（扫码登录、Cookie 管理）
- [ ] 单简历上传管理（PDF/Word）
- [ ] 基础筛选规则（薪资范围、地点关键词、排除关键词）
- [ ] 手动触发筛选和投递（手动点击按钮执行）
- [ ] 投递记录查看（列表展示）

**暂不包含：**
- ❌ 邮件通知（需域名和邮箱服务）

## 九、验收标准

### 功能验收
- 用户可正常注册、登录
- 可成功绑定至少一个招聘平台账号
- 可上传简历并管理
- 可配置筛选规则
- 可执行自动筛选和投递
- 可查看投递记录和统计

### 性能验收
- 页面加载时间 < 3s
- 单次投递操作完成时间 < 10s
- 支持同时运行多个用户的投递任务

- 所有敏感数据加密存储
- API 接口需认证访问
- 无明显安全漏洞
